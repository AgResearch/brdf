Specify whether to use blast evidence to select the reading frame. If you choose to use blast evidence, then the procedure will extract the frame of each hit to any sequence in a protein database. If all hits are in the same frame, then this frame will be used, and only the ORF that overlaps the best hit in this frame will be reported. If there is more than one frame among the hits, this suggests that there may be an indel error in the sequence causing a frameshift - it is suggested you try an algorithm like ESTScan which uses an HMM and is sometimes able to correct indel frameshift errors. In this case the procedure does not use the blast evidence and will simply report translations in all reading frames.

Specify whether the DNA sequences you have entered are spliced transcripts. If they are spliced transcripts, then internal ORFs will only be reported if they begin with a methionine translation start codon. If they are not spliced transcripts - for example genomic DNA - then internal ORFS are not required to start with an internal ORF.
